# v0.18.1 收尾与 v0.19.0 延后计划

> 临时计划文件：当前 GitHub 登录令牌没有 Issue 写权限。权限恢复后，将 v0.19.0 部分迁移到 GitHub Issue，并删除本文件，避免长期文档与计划重复。

## 版本边界

`v0.18.0` 已经发布，不移动或重写现有标签。低风险、行为保持型的 UX 收尾进入 `v0.18.1`；需要重组异步状态或改变覆盖率统计口径的工作进入 `v0.19.0`。

两个版本均不得修改 Prisma Schema、核心业务 API 语义、SSE/NDJSON、WorkHistory、既有 sessionStorage 键或 Excel 格式。WeComCatch 继续作为仓库外的可选本地工具。

## v0.19.0：AI 工作流与前端可测试性

### 开工前闸门

- [ ] 确认 `main`、最近发布标签和 `package.json` 版本一致，工作区干净。
- [ ] 创建并校验数据库备份，记录真实 `dev.db` 的 SHA-256、大小和修改时间。
- [ ] 使用 Node 24 / npm 11 执行 `npm ci`、迁移状态、现有测试、E2E 和构建。
- [ ] 保存录入、复核、反馈、转写和快速评分行为基线。
- [ ] 每个阶段独立提交；出现协议或数据行为变化时停止并单独评审。

### 阶段 1：AI 工作流状态模型

- [ ] 定义“等待 → 校验 → 生成 → 待复核 → 保存 → 完成 / 失败 / 取消”的可辨识联合类型、action 和状态转换表。
- [ ] 为状态转换、失败重试、取消和恢复增加纯 reducer 单元测试。
- [ ] 明确安全持久化字段；File、Blob、音频、API Key 和 Token 禁止进入 `sessionStorage`。
- [ ] 保持旧 `chem-track:nl-input-draft`、`chem-track:feedback-draft` 和 WorkHistory 兼容。
- [ ] 先让课堂录入接入，再让反馈接入；每接入一个工作台先跑完整回归。
- [ ] 不改变现有解析、复核和确认接口语义。

### 阶段 2：录入工作台职责合并

- [ ] 让 Entry reducer 管理输入、解析、复核和确认的显式流程状态，不再只管理标签页。
- [ ] 保留 `useInputWorkspace` 与 `useReviewWorkspace` 中稳定的请求实现，先通过 adapter 接入，禁止整段重写。
- [ ] 统一错误、重试、已恢复、未保存和已完成提示。
- [ ] 验证刷新、切换页面、切换课次、历史恢复和重复确认。
- [ ] 页面组件只负责组合，不承载 SSE、历史恢复和业务判断。

### 阶段 3：录音转写工作台拆分

- [ ] 从 `DiarizeWorkspace` 提取任务 API、录音控制、任务列表、任务详情和结果发送模块。
- [ ] 只持久化引擎和任务 ID；本地 File/Blob 继续在离页前提醒。
- [ ] 保持 `auto` 云端优先提示、`local` 纯本地选择和现有任务目录格式。
- [ ] 保持上传、流式事件、重试、删除及“送入录入”行为。
- [ ] 不修改 FunASR、WeComCatch 外部仓库或迁移任何运行数据。

### 阶段 4：快速评分剩余编排拆分

- [ ] 以 `src/features/quick-score/api.ts` 为边界，按参考数据、课次选择、保存和历史恢复拆分 `useQuickScorePage`。
- [ ] 保留 ref 协调、差异提交、历史恢复和默认分数语义。
- [ ] 先补错误、竞态和恢复测试，再移动逻辑。
- [ ] 不修改业务接口和数据库写入规则。

### 阶段 5：覆盖率口径重建

- [ ] 显式列出需要统计的 `app/api`、`services`、`lib` 和 Feature 逻辑文件，避免只统计运行时碰巧加载的文件。
- [ ] 展示 TSX 继续由 Playwright 负责；复杂 hook、reducer 和 API adapter 必须进入可解释的测试口径。
- [ ] 先记录新口径基线，再决定阈值；不得为了保持旧百分比扩大排除列表。
- [ ] 为 AI reducer、转写模块、快速评分错误分支和 session 恢复增加测试。
- [ ] 发布说明同时记录覆盖范围和百分比。
